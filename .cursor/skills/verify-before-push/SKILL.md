# Push前検証

CIと同様の検証をローカルで実行し、push前に問題を検出する。

## 使用タイミング

- リモートへpushする前に、ビルド・テストが通るか確認したいとき
- プルリクエスト作成前の最終確認
- CIで失敗した原因をローカルで再現・修正したいとき

## 実行手順

リポジトリルートで以下を順に実行する。

### Step 1: テスト

```bash
./gradlew test
```

- 単体テスト・統合テストを実行する
- 失敗した場合は即座に停止し、内容を報告して修正案を提示する

### Step 2: ビルド

```bash
./gradlew build
```

- テストを含めた全体ビルドを実行する
- ビルド失敗時は原因を分析し、修正案を提示する

## エラー時の対応

1. **即座に停止**し、エラー内容を報告する
2. 原因を分析し、修正案を提示する
3. ユーザーの承認を得てから修正する
4. 修正後、**失敗したステップから再開**する

## 結果報告

完了後、次の形式で報告する。

```
## Push前検証 結果

| Step   | 結果 |
|--------|------|
| test   | OK / NG |
| build  | OK / NG |

（エラー時は詳細と修正内容を記載）
```
